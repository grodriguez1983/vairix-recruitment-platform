- **Fecha**: YYYY-MM-DD
- **Decisores**: nombres o roles
- **Relacionado con**: spec.md §N, data-model.md, ADR-XXX, etc.

---

## Contexto

Qué problema estamos resolviendo y **por qué ahora**. Evidencia que
lo motiva (métrica, bug, requerimiento, aparición de nuevo stack).
Restricciones conocidas (tiempo, budget, stack existente, compliance).

No asumir contexto: un lector stateless del futuro debe entender el
problema desde este texto solo.

---

## Decisión

Qué decidimos hacer, enunciado de forma que pueda discutirse punto
por punto. Si hay varios sub-items, numerarlos.

1. …
2. …
3. …

Código, schema, ejemplos donde aclaren. No poner "se discutirá"
o "TBD"; si no se decidió, no es una decisión.
